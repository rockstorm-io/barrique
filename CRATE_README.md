A standard implementation of `barrique` binary serialization format.

`barrique` is a serialization format designed for local out-of-memory
caches, featuring metadata, compression and streaming. Data stored in
the core primitive — region. Regions are LZ4 compressed blocks
of encoded bytes additionally storing small metadata header.

# Example

Most likely you'll start with implementations generated by derive macro:

```rust
use barrique::{Encode, Decode};
use barrique::frame::Frame;

#[derive(Encode, Decode, Clone, Debug, PartialEq)]
struct Bee {
    name: String,
    state: State,
    #[barrique(skip = 2)]
    age: u8,
}

#[derive(Encode, Decode, Clone, Debug, PartialEq)]
#[barrique(tag_repr = "u8")]
enum State {
    Collecting(i32, i32),
    Buzzing {
        sound_level: u8
    },
    Sleeping
}

fn main() {
    let bee = Bee {
        name: "Oh, hey!".to_string(),
        state: State::Sleeping,
        age: 2
    };
    let mut dst = vec![];

    let frame = Frame::new(&mut dst)
        .with_label("A beehive".try_into().unwrap());
    frame.encode(bee.clone()).unwrap();

    let frame: Frame<Bee, _> = Frame::decode(dst.as_slice(), None)
        .expect("Failed to open a frame");

    assert_eq!(frame.get_value(4 * 1024).unwrap(), bee);
}
```

`Frame` is specification-defined format wrapping a stream primitive. Streams
available as `StreamEncoder` and `StreamDecoder`.

# Streaming

`barrique` format was designed to keep buffer memory usage constant, which is
128 KiB (although it is possible to configure if a value to encode is less
than 128 KiB). The size limit is the exact length of two raw regions, previous
region is stored to improve LZ4 compression ratio, which maximum window size is 64 KiB.

In order to utilize streaming, the crate provides wrappers for `std::io::Write` and `std::io::Read`
traits called `CursorWriter` and `CursorReader`. Memory usage will not exceed 192 KiB in these
conditions (128 KiB allocated by a stream primitive and additional 64 KiB allocated by an intermediate
buffer), but in practice this number is much lower due to compression

### Example (requires `std` feature)

```rust,no_run
use barrique::cursor::CursorReader;
use barrique::decode::{get, StreamDecoderBuilder};

use std::error::Error;
use std::fs::File;

fn main() -> Result<(), Box<dyn Error>> {
    let mut file = File::open("dinosaur.bin")?;
    assert_eq!(file.metadata()?.len(), 8 * 1024 * 1024);

    let reader = CursorReader::new(&mut file);
    let mut decoder = StreamDecoderBuilder::new(8 * 1024 * 1024)
        .build(reader)?;

    // That's a large string out there!
    let _ = get::<String>(&mut decoder)?;
    Ok(())
}
```

# `no_std` support

Enabled by default, however, `alloc` crate is required. Use `std` feature to add
features dependent on `std` crate
